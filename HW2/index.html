Asteroid dodger

  • High Concept: My game is a simple dodging game. Dodge all the asteroids to win.
  • Theme: I used the asteroid theme.
  • Mandated Variety: The game is very random. The asteroids all spawn in a random location so it can make the game difficult.
  • Player Experience Goals: I want them to be frustrated with the rng but aslo to enjoy the overall game.
  • Gameplay: The gameplay is essentially a stick figure moving back and forth. The player can use the left and right arrow keys as inputs.
  • Prototype Goal: Digital prototyping
  • Features: The game is pretty simple so far, i would say its set apart from other games due to the difficulty, it can be really easy or very difficult.
  • Setting/Premise: The game is all abstract. It takes place on a alternate version of earth that sees lots of asteroid showers that just so happen to not destroy the earth somehow.
  • Story: My game doesn't have a complling story per-say, it's mainly about a guy who lives alone and like to dodge asteroids. Since is the one passtime he has he's pretty good at it.
  • Target Audience: Anyone that like dodging games and has a few minutes to spare
  • Play Time: Anywhere from a few seconds to a minute or two
  • Strategies: The game is all rng so there's no real strategy
  • Prototype Assets: Did you make your prototype assets from scratch? Did you borrow them? Cite your sources here.
    • All the images used were created by me
  • Prototype Closest Other Game: I borrowed bits and pieces from the different phaser examples. I had a hard time getting things to work without breaking. There was an example for a moving ufo which i got the movement from, i learned how to display text, create a group and collision.
